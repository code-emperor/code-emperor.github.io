了解Git和GitHub

什么是GitHub

GitHub是一个面向开源及私有软件项目的托管平台，它只支持git作为唯一的版本库格式，在GitHub中用户可以十分轻易地找到海量开源代码，用户可以利用源代码在其基础上修改和学习，但为了长时间支持免费库的托管，GitHub的开发者不得不通过付费的私有库来支持免费库项目的运行，所以GitHub不管是托管开源代码还是托管未开源代码都是是个不错的选择。

什么是Git

Git是一个开源的分布式版本控制系统，可以高效的处理从很小到很大的项目版本管理，分布式版本控制系统，对应的是集中式的版本控制。分布式的版本控制就是每个人都可以创建一个独立的代码仓库用于管理，各种版本控制的操作都可以在本地完成。每个人修改的代码都可以推送合并到另外一个代码仓库中。而集中式的版本控制，只有一个中央控制，所有的开发人员都必须依赖于这个代码仓库，每次版本控制的操作也必须链接到服务器才能完成，很多公司喜欢用集中式的版本控制是为了更好的控制代码。如果个人开发，就可以选择Git这种分布式的，多人在本地分工合作，分别推送及合并仓库。至于Git的具体功能，本文就不详细介绍的，感兴趣的同学可以自己上网查阅。

为什么学习GitHub

代码托管

就是我们做的最多的，上传代码，云端备份，你可以想象成一个云盘。

项目管理

GitHub最初是为了开发的管理而生，当然也就具备了项目管理的潜质，特别是与开发密切联系的项目中，它的优势尽显。

GitHub Pages

可以为项目建立静态主页(即GH-pages分支), 就是我们所说的搭建个人网站。

个人简历，找工作，面试加分

一个好的GitHub可以为面试加不少分，让你赢在起跑线上。而且现在越来越多知名公司活跃在GitHub，发布开源库并招募各类人才

注册GitHub账号

点击官方网址：https://github.com,进入到GitHub官网首页，如果已有账号直接登录即可

点击“sign up ”进入到注册界面，如下图所示

点击创建按钮，进入以下界面，（免费的开源仓库任何人都可以访问和复制，收费的私有仓库必须是库主自己和他指定的用户可以访问）

点击continue进入一个用户调查问卷的界面，用户可以直接skip this step（跳过这个步骤）


此时账户已经创建好了，但若想建立仓库还需要邮箱验证，但个别用户会无法收到邮件，这是因为部分邮箱需要设置域名白名单（QQ邮箱），具体设置白名单的方法请百度。

创建GitHub仓库

验证邮箱之后，网页直接跳转到用户新注册账户的主页，如下

点击start a project创立一个仓库，如果要建立web的话仓库名必须这样设置：
用户名.github.io,如果仅仅是托管项目代码，仓库名没有严格要求。


Repository：仓库的意思，即你的项目，你想在GitHub上开源一个项目，那就必须要新建一个repository，你也可以建立多个仓库存储多个项目。
Create new file：在仓库里面创立新文件，也就是创立仓库内容。
Star：star后面的数字意思是收藏你这个项目的人数，进入别人的仓库也会有这个按钮，如果点击就会自动收藏别人的仓库。
Fork：在别人的仓库上点击这个按钮，你自己的账户中会存在一个相同的仓库，也就是克隆，这样做常常因为想要在别人的项目的基础上做些改进，然后应用到自己的项目中去。
Issue：如果你看到别人仓库中的项目代码有缺陷（bug），可以在issue中向别人发送建议和提醒,当然别人也可以通过你的issue向你发送建议和bug提醒。
Pull requests：这个是基于Fork的，如果你在别人的基础上做一些改进，为了让你的改进让更多人收益，于是你把改进合并到你克隆的项目中，向原项目的创建人发送一个Pull requests，
如果原项目创建人觉得你的改进可以的话，他就可以同意你的改进。这就是一个互相学习的的过程。
Wach:关注别人的仓库之后，如果别人的仓库有任何更新，都会第一时间收到别人的项目的更新提醒。

Git初始化及仓库创建和操作

基本信息设置

新建一个文件夹，在文件内部右键Git Bash Here，打开命令行窗口

设置用户名和用户名信箱
在命令行中输入：git config --global user.name ‘GitHub用户名’（用户名不能写错，否则上传之后，仓库将会显示错误）
在命令行中输入：git config --global email.name ’注册的邮箱’
该设置在仓库主页显示谁提交了该文件

初始化一个新的Git仓库

创建文件夹

可以鼠标右击创建文件夹，也可以在命令行中输入‘mkdir’仓库名 的方式来创建文件夹

在文件内初始化Git（创建Git仓库）

分别输入以下命令：cd test：可以进入test文件内部
pwd：查看当前文件目录、git init
可在test文件夹下得到一个.git文件夹（用来存储仓库信息），如果你的电脑不显示，可能是因为你的电脑设置自动隐藏了，在test文件里点击查看，点击最右边弹出的选项，点击查看，最后将隐藏文件显示一栏打上勾.

向仓库中添加文件

创建要添加的文件

在命令行中输入：touch ‘文件名’（创建一个需要向仓库中添加的文件.）git status

将文件提交到暂存区

在命令行中输入：git add ‘文件名’和git status

将暂存区文件提交到GitHub仓库

继续在命令行输入：git commit -m ‘add 文件名‘和git status’

编辑和修改文件内容

在命令行中输入：vi 文件名，进入文件中编写代码（注意：ESC键是模式的转化，：wq再回车是退出并保存，如果你不会语法建议用其他编辑器编写或者看相关书籍和教程）

继续用上面1、2、3步骤将文件先传到暂存区，再传到仓库中

删除仓库文件

在命令行中输入：rm -rf 文件名（也可以直接用鼠标删除文件）
Git rm 文件名
Git commit -m’描述

Git管理远程仓库

上一章我们都是在本地进行操作，在这个过程中对云端GitHub仓库没有丝毫影响，在现实生活中，公司的项目开发是由多人合作完成的，为了方便代码的管理，同一个项目的开发者们会在本地上传到云端同一个GitHub仓库中，实现代码的共享集中化管理

那么如何将本地仓库提交到远程呢？这个时候就需要运用到git push命令了

Git的克隆操作

仓库地址：

命令行中继续输入： git config –list检查初始化
git clone 仓库地址

将本地仓库推到远端仓库

本节和上文内容一样，先建立文件、将文件推送到本地暂存区，再推送到本地仓库，最后将本地仓库推送到远端，下面是具体操作
命令行输入：touch ‘文件名’、git add ‘文件名’、 git commit -m ‘add 文件名‘、git push
在推送的过程中需要输入GitHub账号和密码.
注意：有些人在访问时会出现错误，请检查有没有初始化，还有人以上步骤没有出错，但由于之前用了其他GitHub账号，导致推送时读取了其他GitHub账号信息，出现了403报错,可以用编辑器修改.gif/config文件下的URL=https://用户名：密码@github.com/用户名.仓库名.git，也可以删除相关系统证书.
